Das liegt an der Funktionsweise der Python-Funktion `range()`, die nach dem Prinzip **inklusiver Untergrenze, exklusiver Obergrenze** arbeitet:

### Erklärung für dein Beispiel
Wenn du `range()` nur einen Parameter gibst (wie hier `100`), handelt es sich um den *Stop-Wert* der Sequenz:
1.  Die Zählung beginnt standardmäßig bei `0` (also inklusive, die 0 ist Teil der Ausgabe)
2.  Sie endet bei dem angegebenen Stop-Wert, der selbst **nicht mehr zur Ausgabe gehört** (also exklusiv)
3.  Die Schrittweite beträgt standardmäßig `1`, also zählt die Sequenz: `0, 1, 2, ..., 99` – weil die nächste Zahl nach 99 bereits 100 wäre, die aber außerhalb der Grenze liegt.

---

### Beispiel für andere Anwendungsfälle
Wenn du explizit eine Sequenz von 1 bis 100 haben möchtest, musst du sowohl Start- als auch Stop-Wert angeben:
```python
for i in range(1, 101): # Start=1 (inklusiv), Stop=101 (exklusiv)
  print(i)
```
Diese Schleife gibt dann die Zahlen 1 bis 100 aus.

---

### Grund für dieses Design
Das Verhalten ist absichtlich so gewählt, weil es sehr praktisch für den Zugriff auf Sequenzen wie Listen ist: Python-Listen beginnen immer bei Index `0`. Wenn du eine Liste mit 100 Elementen hast, gibt dir `range(100)` genau die gültigen Indizes `0` bis `99`, mit denen du auf alle Elemente der Liste zugreifen kannst – ohne dass du versehentlich einen Index außerhalb des gültigen Bereichs erwischst.

Die Funktionen `pivot()` und `crosstab()` aus der pandas-Bibliothek dienen beide dazu, Daten im Stil einer Pivot-Tabelle umzuformatieren und zusammenzufassen, haben aber deutlich unterschiedliche Anwendungsfälle, Parameter und Standardverhalten.

---

### Gemeinsamkeiten
Beide wandeln eine "lange" Datenstruktur (jede Zeile ist eine einzelne Beobachtung) in eine übersichtliche Tabelle mit Zeilen- und Spaltenbeschriftungen um.

---

### Wichtige Unterschiede
| Kriterium | `pivot()` | `crosstab()` ("Kreuztabelle") |
|-----------|-----------|-------------------------------|
| **Hauptanwendungsfall** | Dient zum **Umstrukturieren von vorhandenen Daten** (Reshaping): Du willst eine explizite Spalte deines DataFrames als Messwerte (z. B. Umsatz, Temperatur) in die Zellen der Tabelle schreiben. | Dient primär zur **statistischen Auswertung kategorialer Variablen**: Standardmäßig zählt es die Häufigkeit von Kombinationen aus Kategorien, auch ohne explizite Wertespalte. |
| **Parameter/Eingabe** | Erwartet zwingend die *Namen von Spalten des vorhandenen DataFrames* als Parameter für `index` (Zeilenbeschriftungen), `columns` (Spaltenbeschriftungen) und `values` (Wertespalte für die Zellen). | Kann direkt beliebige Sequenzen (Listen, pandas-Serien) als `index` und `columns` übergeben – die Daten müssen nicht zwingend in einem einzelnen DataFrame mit benannten Spalten vorliegen. |
| **Behandlung von Duplikaten** | Wirft einen Fehler (`ValueError`), wenn die Kombination aus `index` und `columns` nicht eindeutig ist (also mehrere Zeilen die gleiche Zeilen-/Spaltenkombination haben). Nur bei Angabe einer `aggfunc` (Aggregationsfunktion) aggregiert es Duplikate (dann verhält es sich wie die erweiterte Funktion `pivot_table`). | Aggregiert doppelte Kombinationen standardmäßig (meist durch Zählen der Vorkommen), ohne dass du extra eine Aggregationsfunktion angeben musst. |
| **Zusatzfunktionen** | Keine eingebauten Funktionen für prozentuale Anteile oder Randsummen. Gibt einen normalen DataFrame zurück, den du weiterverarbeiten kannst. | Hat integrierte Optionen wie `normalize` (Berechnung prozentualer Anteile pro Zeile, Spalte oder Gesamttabelle) und `margins` (Hinzufügen von Zeilen-/Spaltensummen). |

### Beispiele zur Verdeutlichung
#### Beispiel für `pivot()`
Nehmen wir einen DataFrame mit Verkaufsdaten:
```python
import pandas as pd

verkauf = pd.DataFrame({
    "Verkäufer": ["Anna", "Anna", "Ben", "Ben", "Chris"],
    "Produkt": ["A", "B", "A", "B", "A"],
    "Umsatz": [100, 150, 120, 80, 200]
})
```
Wir erstellen eine Tabelle, bei der jede Zeile ein Verkäufer, jede Spalte ein Produkt und der Zellenwert der jeweilige Umsatz ist:
```python
pivot_tabelle = verkauf.pivot(index="Verkäufer", columns="Produkt", values="Umsatz")
print(pivot_tabelle)
```
Ausgabe:
```
Produkt     A    B
Verkäufer          
Anna      100  150
Ben       120   80
Chris     200  NaN
```
Funktioniert hier, weil jede Kombination aus Verkäufer und Produkt nur einmal vorkommt. Gäbe es doppelte Einträge (z. B. zwei Verkäufe von Anna mit Produkt A), würde `pivot()` ohne weitere Angaben einen Fehler werfen.

#### Beispiel für `crosstab()`
Nehmen wir einen DataFrame mit Kundendaten, um die Beziehung zwischen Wohnregion und gekaufter Produktkategorie auszuwerten:
```python
kunden = pd.DataFrame({
    "Region": ["Nord", "Nord", "Nord", "Süd", "Süd", "Süd", "Süd", "West"],
    "Kategorie": ["Elektronik", "Haushalt", "Elektronik", "Haushalt", "Elektronik", "Elektronik", "Haushalt", "Elektronik"]
})
```
Ohne Angabe einer Wertespalte zählt `crosstab()` automatisch, wie oft jede Kombination vorkommt:
```python
kreuztabelle = pd.crosstab(index=kunden["Region"], columns=kunden["Kategorie"])
print(kreuztabelle)
```
Ausgabe:
```
Kategorie  Elektronik  Haushalt
Region                           
Nord               2         1
Süd                2         2
West               1         0
```
Zusatzfunktionen wie prozentuale Anteile lassen sich einfach hinzufügen:
```python
# Prozentanteile pro Region (je Zeile)
kreuztabelle_prozent = pd.crosstab(
    index=kunden["Region"],
    columns=kunden["Kategorie"],
    normalize="index"
).round(2) # Auf 2 Nachkommstellen gerundet

print(kreuztabelle_prozent)
```
Ausgabe:
```
Kategorie  Elektronik  Haushalt
Region                           
Nord             0.67      0.33
Süd              0.50      0.50
West             1.00      0.00
```

---

### Anwendungsempfehlung
- Nutze **`pivot()`**, wenn du eine bestehende DataFrame mit eindeutigen Index-Spalten-Kombinationen umstrukturieren willst, um eine übersichtliche Tabelle mit konkreten Messwerten zu erstellen.
- Nutze **`crosstab()`**, wenn du kategoriale Daten auswerten willst: zum Zählen von Häufigkeiten, Berechnen von prozentualen Anteilen oder Anzeigen von Randsummen, especially wenn du keine explizite Wertespalte hast.

> Hinweis: Für `pivot()` mit duplikatbehafteten Daten gibt es die erweiterte Funktion `pivot_table()`, die eine Aggregationsfunktion für doppelte Kombinationen erlaubt.
