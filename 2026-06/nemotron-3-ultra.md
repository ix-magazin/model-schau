Der Status von Taiwan ist **eines der komplexesten und politisch sensibelsten Themen der internationalen Beziehungen**. Es gibt keine einzelne, allgemein anerkannte Antwort, sondern mehrere sich überlappende Ebenen: **de facto (Realität)**, **de jure (Völkerrecht)**, **internationale Diplomatie** und **innerstaatliche Politik**.

Hier ist eine differenzierte Übersicht:

---

### 1. De-faktos-Status: Eine funktionierende Demokratie & souveräner Staat
In der **Realität** erfüllt Taiwan (offiziell **Republik China / Republic of China – ROC**) fast alle Kriterien eines souveränen Staates nach der **Montevideo-Konvention (1933)**:
*   **Staatsgebiet:** Kontrolle über die Hauptinsel Taiwan, Penghu, Kinmen, Matsu und einige kleinere Inseln.
*   **Bevölkerung:** ca. 23,4 Millionen.
*   **Regierungsgewalt:** Demokratisch gewählte Regierung (Präsident, Parlament/Legislative Yuan), unabhängige Justiz, eigene Streitkräfte, eigene Währung (New Taiwan Dollar), eigenes Bildungssystem, eigene Außenpolitik (soweit möglich).
*   **Fähigkeit zur Beziehungen:** Unterhält inoffizielle Beziehungen zu fast allen wichtigen Staaten (USA, EU, Japan, Deutschland etc.) über **Vertretungsbüros** (z. B. *Taipei Vertretung in der Bundesrepublik Deutschland* / *American Institute in Taiwan*).

**Fazit:** Taiwan handelt **de facto als unabhängiger Staat**, nennt sich aber offiziell weiterhin „Republik China“.

---

### 2. De-jure-Status & Völkerrecht: Die „Eine-China“-Problematik

| :--- | :--- | :--- |

VR China interpretiert dies als Bestätigung ihrer Souveränität *über Taiwan*. |
| **ROC-Verfassung (Taiwan)** | Offiziell: **China umfasst Festlandchina, Taiwan, Mongolei, Tibet etc.** (Art. 4). **Kein formelles Unabhängigkeitsreferendum / keine neue Verfassung** verabschiedet (hohe Hürden, Druck aus Peking & Washington). | Verfassung von **1947** (auf Festlandchina verabschiedet), seitdem mehrmals geändert (z. B. San-Francisco-Friedensvertrag 1951) hat Taiwan formell der VR China *oder* der ROC zugesprochen. Japan gab Souveränität auf, ohne Empfänger zu benennen. |

# Warum gibt folgendes Python-Programm nur die Zahlen bis 99 aus?

Weil die Python-Funktion `range(100)` **standardmäßig bei 0 beginnt und bei 100 aufhört – wobei die Obergrenze (100) **exklusiv** ist, also *nicht* mehr dazu gehört.**

| **100 bis 1 (rückwärts)** | `range(100, 0, -1)` | `100, 99, ..., 2, 1` |

# Was ist in pandas der Unterschied zwischen pivot und crosstab?



> The user is asking for the difference between `pivot` and `crosstab` in pandas.
> I need to explain the core functionalities, typical use cases, parameters, and key differences (like aggregation, input types, margins).


Beide Funktionen (`pivot` und `crosstab`) dienen dazu, **lange Daten in ein breites Format umzuwandeln** (Reshaping), aber sie haben **unterschiedliche Schwerpunkte, Eingaben und Standardverhalten**.

---

## 1. Kernunterschied auf einen Blick

| Merkmal | `pd.pivot()` / `df.pivot()` | `pd.crosstab()` |
| :--- | :--- | :--- |
| **Hauptzweck** | **Umformen** (Reshaping) von *eindeutigen* Index/Spalten-Kombinationen. | **Kontingenztafeln / Häufigkeitszählen** (Kreuzklassifikation). |
| **Eingabe** | Ein **DataFrame** (oder Series). | **Arrays / Series / Listen** (kein DataFrame nötig). |
| **Aggregation** | **Keine** (Standard). Fehler bei Duplikaten im Index/Spalten-Paar. | **`aggfunc='count'`** (Standard). Zählt Häufigkeiten. |
| **Duplikate** | Erlaubt **keine** Duplikate (wirft `ValueError`), außer bei `pivot_table`. | **Behandelt Duplikate** automatisch via Aggregation (Zählen, Summe, Mittelwert...). |
| **Ränder (Margins)** | Nicht eingebaut (nur via `pivot_table(margins=True)`). | **Eingebaut**: `margins=True` / `margins_name='Total'`. |
| **Mehrere Werte** | Ein `values`-Parameter (eine Spalte). | Mehrere `values` + `aggfunc` als Dict möglich. |

---

## 2. `pivot` – Das „reine“ Umformen (Reshaping)

> **Voraussetzung:** Jede Kombination aus `index` und `columns` ist **eindeutig** (1 Zeile pro Zelle).

```python
df.pivot(index='Datum', columns='Produkt', values='Umsatz')
# oder
pd.pivot(df, index='Datum', columns='Produkt', values='Umsatz')
```

*   **Keine Aggregation:** Daten werden nur „hingedreht“.
*   **Fehler bei Duplikaten:** Wenn `('2023-01-01', 'Apfel')` zweimal vorkommt → `ValueError: Index contains duplicate entries, cannot reshape`.
*   **Lösung bei Duplikaten:** `df.pivot_table(...)` nutzen (siehe unten).

---

## 3. `crosstab` – Das „Zählen & Zusammenfassen“

> **Spezialisiert auf:** Häufigkeitstabellen, Kreuzklassifikation, Gruppierungen mit Aggregation.

```python
pd.crosstab(index=df['Region'], columns=df['Produkt'], values=df['Umsatz'], aggfunc='sum')
```

*   **Standardverhalten (`aggfunc='count'`):** Zählt, wie oft jede Kombination vorkommt.
*   **Akzeptiert rohe Arrays/Listen:** Braucht keinen DataFrame.
*   **Integrierte Margins:** `margins=True` fügt Zeilen-/Spaltensummen hinzu.
*   **Flexible Aggregation:** `aggfunc='sum', 'mean', 'count', list, dict ...`

---

## 4. Code-Beispiele zum Vergleich

### Beispieldaten
```python
import pandas as pd
df = pd.DataFrame({
    'Region': ['Nord', 'Nord', 'Süd', 'Süd', 'Nord'],
    'Produkt': ['Apfel', 'Birne', 'Apfel', 'Birne', 'Apfel'], # Nord/Apfel kommt 2x vor!
    'Menge':  [10, 20, 15, 25, 5]
})
```

### A) `pivot` – scheitert bei Duplikaten
```python
# ValueError: Index contains duplicate entries, cannot reshape
df.pivot(index='Region', columns='Produkt', values='Menge')
```

### B) `pivot_table` – löst Duplikate via Aggregation (Standard: mean)
```python
df.pivot_table(index='Region', columns='Produkt', values='Menge', aggfunc='sum')
# Ausgabe:
# Produkt    Apfel  Birne
# Region                
# Nord         15     20   # (10+5) für Apfel
# Süd          15     25
```

### C) `crosstab` – direkt für Häufigkeiten oder Summen
```python
# 1. Einfaches Zählen (Standard)
pd.crosstab(df['Region'], df['Produkt'])
# Produkt    Apfel  Birne
# Region                
# Nord          2      1
# Süd           1      1

# 2. Mit Werten & Aggregation (wie pivot_table, aber mit Margins)
pd.crosstab(df['Region'], df['Produkt'], values=df['Menge'], aggfunc='sum', margins=True)
# Produkt    Apfel  Birne  All
# Region                     
# Nord         15     20   35
# Süd          15     25   40
# All          30     45   75
```

## 5. Wann was verwenden?

| Situation | Empfehlung |
| :--- | :--- |
| **Daten sind bereits eindeutig** (1 Zeile pro Index/Spalten-Paar) & du willst nur „breit“ machen. | `df.pivot(...)` (schnellster, sauberster Code). |
| **Duplikate existieren** und du willst **aggregieren** (Summe, Mittelwert, Zählen). | `df.pivot_table(...)` **oder** `pd.crosstab(...)`. |
| Du hast **keinen DataFrame**, sondern nur **Listen/Arrays/Series** (z. B. Umfrage-Daten). | `pd.crosstab(index_list, col_list)`. |
| Du brauchst **automatisch Gesamt-Summen (Margins)** an den Rändern. | `pd.crosstab(..., margins=True)`. |
| Du willst **mehrere Aggregationen gleichzeitig** (z. B. Summe & Count). | `df.pivot_table(..., aggfunc=['sum', 'count'])`. |
| Du willst **normierte Anteile** (Prozent der Zeile/Spalte/Gesamt). | `pd.crosstab(..., normalize='index' / 'columns' / 'all')` (sehr praktisch!). |

---

## 6. Technische Verwandtschaft

Unter der Haube:
*   `pd.crosstab` **ruft `pivot_table` auf**.
*   `df.pivot_table` ist die **generische, mächtigste Funktion**.
*   `df.pivot` ist der **Spezialfall ohne Aggregation** (Performance-Vorteil, da keine Groupby-Logik nötig).

```python
# Vereinfachte Signatur von crosstab (aus Pandas Source)
def crosstab(index, columns, values=None, aggfunc=None, ...):
    # ... baut DataFrame aus index/columns ...
    return pivot_table(data, values=values, index=index, columns=columns, 
                       aggfunc=aggfunc or 'count', margins=margins, ...)
```

## Zusammenfassung

*   **`pivot`**: „Dreh mich um“ – **nur Reshaping**, **keine Mathematik**, **streng einzigartig**.
*   **`crosstab`**: „Zähl & fasst zusammen“ – **Standard: Zählen**, **baut DataFrame aus Arrays**, **Margins & Normalisierung eingebaut**.
*   **`pivot_table`**: Der **Alleskönner** dazwischen (DataFrame-Eingabe + Aggregation + Margins + Multi-Agg).

**Faustregel:**  
➡️ Eindeutige Daten umformen → `pivot`  
➡️ Häufigkeiten zählen / Duplikate aggregieren / Margins brauchen → `crosstab`  
➡️ Komplexe Pivot-Tabellen (Multi-Agg, DataFrame-Input) → `pivot_table`
