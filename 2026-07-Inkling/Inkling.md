Das Programm gibt die Zahlen `0` bis `99` aus, weil die Python-Funktion `range()` **am oberen Ende exklusiv** arbeitet.

```python
for i in range(100): 
    print(i)
```

Hier passiert Folgendes:

- `range(100)` erzeugt die Sequenz von `start=0` (Standard) bis `stop=100`, **ohne die 100 selbst einzuschließen**.
- Das ergibt genau die Werte:

$$0, 1, 2, \dots, 99$$

Insgesamt sind das $100 - 0 = 100$ Zahlen.

Wenn du bis einschließlich `100` zählen willst, musst du den oberen Wert um `1` erhöhen:

```python
for i in range(101):  # 0 bis einschließlich 100
    print(i)
```

Oder explizit mit Start- und Endwert:

```python
for i in range(0, 101):
    print(i)
```

**Zusammenfassung:** `range(n)` liefert immer die Zahlen von `0` bis `n-1`.

Der Kernunterschied: **`pivot`** formt Daten von lang nach breit um (Reshape), während **`crosstab`** eine **Kreuztabelle mit Aggregation** (meist Häufigkeiten) erstellt.

<details>
<summary><b>📊 Kernunterschied auf einen Blick</b></summary>

| Merkmal | `df.pivot()` | `pd.crosstab()` |
|---|---|---|
| **Zweck** | Daten umstrukturieren (Wide-Format) | Häufigkeits-/Kontingenztabelle erstellen |
| **Aggregation** | Keine (wirft Fehler bei Duplikaten) | Ja, standardmäßig `count` (`len`) |
| **Eingabe** | `DataFrame` | `Series`, Arrays oder `DataFrame`-Spalten |
| **Doppelwerte** | `ValueError`, wenn Kombination nicht eindeutig | Wird aggregiert (z. B. summiert/gezählt) |
</details>

<details>
<summary><b>🔧 <code>pivot</code> – Umformen ohne Aggregation</b></summary>

`pivot` erwartet, dass die Kombination aus `index` und `columns` **eindeutig** ist.

```python
import pandas as pd

df = pd.DataFrame({
    'Datum': ['2024-01', '2024-01', '2024-02', '2024-02'],
    'Kategorie': ['A', 'B', 'A', 'B'],
    'Wert': [10, 20, 30, 40]
})

# Lange Form → Breite Form
df_pivot = df.pivot(index='Datum', columns='Kategorie', values='Wert')
# Ergebnis:
# Kategorie   A   B
# Datum            
# 2024-01   10  20
# 2024-02   30  40
```

Wenn es bei `('2024-01', 'A')` zwei Zeilen gäbe, würde `pivot` einen `ValueError` werfen. Für Aggregation nutzt man dann **`pivot_table`**:

```python
df.pivot_table(index='Datum', columns='Kategorie', values='Wert', aggfunc='mean')
```
</details>

<details>
<summary><b>📈 <code>crosstab</code> – Kreuztabelle mit Aggregation</b></summary>

`crosstab` zählt standardmäßig, wie oft jede Kombination vorkommt.

```python
# Einfacher Fall: Häufigkeiten zweier Kategorien
ct = pd.crosstab(df['Datum'], df['Kategorie'])
# Datum     A  B
# 2024-01  1  1
# 2024-02  1  1

# Mit Werten und anderer Aggregation
ct_sum = pd.crosstab(
    df['Datum'], 
    df['Kategorie'], 
    values=df['Wert'], 
    aggfunc='sum'
)
```

`crosstab` akzeptiert auch `normalize` für Prozentwerte und kann mit `margins` (Randsummen) arbeiten.
</details>

<details>
<summary><b>✅ Wann was verwenden?</b></summary>

- Nutze **`pivot`** (oder `pivot_table`), wenn du bereits numerische Werte hast und diese nur in eine andere Form bringen willst (z. B. Zeitreihen als Spalten).
- Nutze **`crosstab`**, wenn du die **Verteilung** von Kategorien analysieren willst (z. B. „Wie oft kommt Kombination X vor?“) oder wenn du explizit aggregieren musst.
</details>

Zusammengefasst: `pivot` ist ein **Reshape-Tool**, `crosstab` ein **Statistik-Tool** für Kontingenztabellen.
