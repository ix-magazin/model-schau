> 
> I need to explain this behavior of the `range()`function in Python.


Das Python-Programm gibt die Zahlen von 0 bis 99 aus, weil dies genau so von der Funktion `range()` in Python vorgesehen ist.

## Die Funktionsweise von `range()`

In Python erzeugt die Funktion `range(stop)` eine Sequenz von Zahlen, die **bei 0 beginnt** und **bis zum angegebenen Endwert (stop) reicht, diesen aber nicht einschließt**.

Wenn Sie `range(100)` schreiben, generiert Python die Zahlen:
$$0, 1, 2, 3, \dots, 99$$

Das bedeutet:
1. Der Startwert ist standardmäßig `0`.
2. Der Endwert ist exklusiv (exclusive), die Zahl `100` selbst wird also nicht mehr ausgegeben.
3. Es werden genau 100 Zahlen ausgegeben (von 0 bis 99).

## Wie Sie die 100 erreichen

Wenn Sie möchten, dass die Zahlen von 0 bis 100 (inklusive 100) ausgegeben werden, müssen Sie den Endwert auf `101` setzen:

```python
for i in range(101):
  print(i)
```

Wenn Sie die Zahlen von 1 bis 100 ausgeben möchten, können Sie einen Startwert angeben:

```python
for i in range(1, 101):
  print(i)
```

Hier ist `range(start, stop)`, wobei der Startwert `1` inklusiv ist und der Endwert `101` exklusiv ist.

Sowohl `pivot` als auch `crosstab` sind Funktionen in `pandas`, die der Umstrukturierung und Zusammenfassung von Daten dienen. Der Hauptunterschied liegt in ihrem **Zweck**, der **Daten-Standardaggregation** und der **Art der Eingabedaten**. 

Oft wird umgangssprachlich von "pivot" gesprochen, wenn eigentlich `pivot_table` gemeint ist. Daher betrachten wir hier beide Varianten.

## 1. `pivot` (und `pivot_table`)

- **`df.pivot()`**: Dient rein der Umformung von Daten (Reshaping). Es wandelt Long-Daten in Wide-Daten um (Werte einer Spalte werden zu neuen Spaltenüberschriften). Es führt **keine Aggregation** durch. Wenn es doppelte Einträge für die Kombination aus Index und Spalten gibt, wirft die Funktion einen Fehler.
- **`pd.pivot_table()`**: Dient der Aggregation von Daten. Wenn es doppelte Einträge gibt, wird standardmäßig der **Mittelwert** (`mean`) berechnet, man kann aber jede andere Aggregationsfunktion (Summe, Anzahl etc.) über den Parameter `aggfunc` festlegen.

## 2. `crosstab`

- **`pd.crosstab()`**: Berechnet eine einfache Kreuztabelle (Kontingenztabelle), die standardmäßig die **Häufigkeiten** (Anzahl der Vorkommnisse) der Kombinationen von zwei oder mehr Faktoren anzeigt. 
- Es ist im Grunde eine spezialisierte Version von `pivot_table` (oder `groupby` + `size`), die hauptsächlich für statistische Analysen verwendet wird, um Zusammenhänge zwischen kategorialen Variablen zu untersuchen.

## Hauptunterschiede im Detail

| Merkmal | `pivot` / `pivot_table` | `crosstab` |
| :--- | :--- | :--- |
| **Aufrufart** | DataFrame-Methode (`df.pivot_table()`) oder Top-Level-Funktion (`pd.pivot_table()`) | Top-Level-Funktion (`pd.crosstab()`) |
| **Eingabedaten** | Nimmt ein DataFrame und Spaltennamen entgegen. | Nimmt Array-ähnliche Objekte (Listen, Series, Arrays) entgegen, die nicht zwingend aus demselben DataFrame stammen müssen. |
| **Standard-Aggregation** | `pivot`: keine (fehler bei Duplikaten)<br>`pivot_table`: Mittelwert (`mean`) | Anzahl / Häufigkeit (`count`) |
| **Zusätzliche Features** | Berechnung von Randsummen (`margins=True`) möglich. | Berechnung von Randsummen (`margins=True`) und Prozentangaben (`normalize=True`) oft einfacher integriert. |
| **Hauptzweck** | Umstrukturierung und Zusammenfassung bestehender DataFrames. | Schnelle Erstellung von Häufigkeitstabellen für Kategorien. |
