# Warum `range(100)` nur bis 99 zählt

Das Verhalten ist **kein Fehler**, sondern liegt an der Definition der eingebauten `range()`-Funktion in Python.

## Die kurze Erklärung

`range(100)` erzeugt eine Zahlenfolge von **0 bis 99** – die Zahl `100` selbst ist **nicht** Teil der Sequenz.

Das liegt an zwei Regeln von `range()`:

1. **Startwert ist standardmäßig $0$** (wenn nur ein Argument übergeben wird).
2. **Die obere Grenze ist exklusiv** – sie wird *nicht* mit ausgegeben.

Man nennt das ein **halboffenes Intervall**: $[0, 100)$

---

## Veranschaulichung

| Aufruf | Erzeugte Zahlen | Anzahl |
|--------|-----------------|--------|
| `range(100)` | $0, 1, 2, \dots, 99$ | $100$ |
| `range(1, 100)` | $1, 2, 3, \dots, 99$ | $99$ |
| `range(0, 100, 1)` | $0, 1, 2, \dots, 99$ | $100$ |

Die allgemeine Syntax lautet:
```python
range(start, stop, step)
```
Wobei `stop` **exklusiv** ist.

---

## Warum macht Python das so?

Das hat historische und praktische Gründe:
- **Zero-Indexing:** Listen und Strings in Python beginnen bei Index $0$. `range(n)` passt dann genau zur Länge einer Liste mit $n$ Elementen:
  ```python
  meine_liste = [10, 20, 30]  # Länge 3
  for i in range(len(meine_liste)):
      print(meine_liste[i])  # i läuft über 0, 1, 2
  ```
- **Saubere Mathematik:** Die Anzahl der Elemente ist einfach `stop - start`. Bei `range(100)` sind es $100 - 0 = 100$ Zahlen.

---

## Wie du bis 100 (oder 1–100) ausgibst

Wenn du die `100` mit dabei haben willst:

```python
# Zahlen 0 bis 100 (also 101 Zahlen)
for i in range(101):
    print(i)
```

```python
# Zahlen 1 bis 100 (ohne die 0)

    print(i)
```

# `pivot` vs. `crosstab` in pandas

Beide Funktionen dienen dazu, Daten von einem **„langen“** (long) in ein **„breites“** (wide) Format zu überführen – also eine Matrix/Sprengtabelle zu erzeugen. Sie unterscheiden sich aber grundlegend in **Eingabe, Standardverhalten und Aggregation**.

---

## 🔑 Kurzüberblick

| Merkmal | `df.pivot()` | `pd.crosstab()` |
|---------|--------------|-----------------|
| **Art** | Methode auf einem DataFrame | eigenständige pandas-Funktion |
| **Eingabe** | Spaltennamen des DataFrames | array-ähnliche Objekte (Series, Listen) |
| **Standard-Aktion** | nimmt *vorhandene* Werte (keine Aggregation) | zählt Häufigkeiten (Counts) |
| **Duplikate?** | ❌ wirft `ValueError` bei Mehrfach-Einträgen | ✅ aggregiert sie (z. B. durch Zählen) |
| ** margins (Summen)** | nicht eingebaut | `margins=True` liefert Zeilen/Spalten-Totale |

---

## 1. `DataFrame.pivot()` – reines Reshaping

`pivot` ordnet vorhandene Werte um. Es wird **keine Berechnung** durchgeführt – die Tabelle wird nur „umgeklappt“.

```python
import pandas as pd

df = pd.DataFrame({
    'Person': ['A', 'B', 'A', 'B'],
    'Month':  ['Jan', 'Jan', 'Feb', 'Feb'],
    'Sales':  [10, 20, 30, 40]
})

result = df.pivot(index='Person', columns='Month', values='Sales')
print(result)
```

Ergebnis:
```
Month   Jan  Feb
Person          
A       10   30
B       20   40
```

⚠️ **Wichtig:** Gibt es für ein `(Person, Month)`-Paar **mehrere Zeilen**, schlägt `pivot` fehl:
```python
# ValueError: Index contains duplicate entries, cannot reshape
```
Dafür bräuchte man `pivot_table` (siehe unten).

---

## 2. `pd.crosstab()` – Kreuztabelle / Häufigkeiten

`crosstab` nimmt zwei (oder mehr) Kategorien und zählt standardmäßig, **wie oft jede Kombination vorkommt**.

```python
pd.crosstab(df['Person'], df['Month'])
```

Ergebnis (reine Zählung):
```
Month   Feb  Jan
Person          
A         1    1
B         1    1
```

Mit `values` + `aggfunc` kann `crosstab` aber auch aggregieren (z. B. Summen bilden), genau wie eine Pivot-Tabelle:
```python
pd.crosstab(df['Person'], df['Month'], 
            values=df['Sales'], aggfunc='sum')
```

Und mit Summenzeilen/­spalten:
```python
pd.crosstab(df['Person'], df['Month'], margins=True)
```

---

## Wann nutze ich was?

- **`pivot`**, wenn deine Daten schon „sauber“ sind (jede Index-Spalten-Kombination exakt einmal vorkommt) und du nur ein bestehendes Feld als Matrix anordnen willst.
- **`crosstab`**, wenn du eine **Kontingenztabelle / Häufigkeitsauszählung** erstellen willst oder mit rohen Arrays/Series arbeitest, die nicht zwingend in einem DataFrame liegen.

<details>
<summary><strong>🧩 Zusatz: Wo passt <code>pivot_table</code> rein? (Klick zum Aufklappen)</strong></summary>

Viele Nutzer verwechseln `pivot` mit `pivot_table`. Der wichtigste Unterschied:

- `df.pivot()` → **keine** Aggregation, Duplikate verboten.
- `df.pivot_table()` → aggregiert bei Duplikaten (Standard: Mittelwert), kennt `margins=True` und ist funktional näher an `crosstab` mit `aggfunc`.

```python
# Bei Duplikaten die Summe bilden:
df.pivot_table(index='Person', columns='Month', 
               values='Sales', aggfunc='sum')
```

`crosstab` ist im Grunde eine Spezialisierung auf Zählungen, während `pivot_table` das allgemeinere Aggregationswerkzeug ist.
